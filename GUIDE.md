# Read Later 使い方ガイド

Read Later は、気になる記事を保存・整理し、AIが自動で情報を集めてくれるパーソナルツールです。

## 1. はじめに — アカウントを作る

### 初回アクセス

1. ブラウザで Read Later のURLにアクセスします
2. 「**新規登録**」をタップ
3. 以下を入力します：
   - **表示名** — アプリ内で表示される名前（日本語OK）
   - **ユーザーID** — ログインに使うID（英数字と`_`のみ、3〜20文字）
   - **パスワード** — 6文字以上
4. 「**登録**」をタップ

> **ポイント**: メールアドレスは不要です。ユーザーIDとパスワードだけで登録できます。

### ログイン

次回以降は、ユーザーIDとパスワードでログインします。
ログイン状態は30日間保持されます。

## 2. 記事を保存する

### PCから保存する

画面上部にある入力欄にURLを貼り付けて「**追加**」ボタンを押します。

```
https://example.com/interesting-article
```

自動でタイトル・サイト名・本文が取得されます。

### スマートフォンから保存する（iOSショートカット）

Safariや他のアプリの「共有」ボタンから、ワンタップで保存できます。
設定方法は [10. スマートフォンから使う](#10-スマートフォンから使う) を参照してください。

## 3. 記事を読む・管理する

### 記事カードの見方

保存した記事は一覧でカード表示されます。各カードには以下の情報があります：

- **タイトル** — タップすると元の記事をブラウザで開きます
- **サイト名** — 記事の出典元
- **保存からの経過時間** — 「3時間前」「2日前」など
- **注目度バッジ** — はてなブックマーク数などに基づくスコア

### ボタンの説明

| ボタン | 機能 |
|--------|------|
| ✓ **既読** | タップすると「読了」に変わり、記事が薄く表示されます。もう一度押すと未読に戻ります |
| ★ **お気に入り** | 大事な記事にマーク。お気に入りタブで絞り込めます |
| ✎ **メモ** | 記事に自分のメモを追加できます |
| # **タグ** | 記事にタグを付けて分類できます |
| × **削除** | 記事を削除します（確認ダイアログが出ます） |
| **読む** | 広告のない読みやすい形式で本文を表示します（リーダービュー） |

### 既読の自動確認

記事のリンクをタップして外部サイトを見た後、アプリに戻ると「**読了にしますか？**」と聞かれます。
「読了」を押せばワンタップで既読にできます。

### 並び替え

画面上部の「**新着**」「**注目**」ボタンで並び替えができます。

- **新着** — 新しく保存した順（デフォルト）
- **注目** — 注目度スコアが高い順（はてブ数・HNポイント・信頼ドメインを考慮）

## 4. タグで整理する

### タグを付ける

1. 記事の **#** ボタンをタップ
2. 既存のタグから選ぶか、新しいタグを作成
3. 色を選んで「**追加**」
4. 「**保存**」で確定

### タグで絞り込む

画面下部の「**タグ**」タブをタップすると、タグ一覧が表示されます。
タグをタップするとそのタグが付いた記事だけが表示されます。

---

## 5. メモを残す

1. 記事の **✎** ボタンをタップ
2. テキストを入力
3. 「**保存**」をタップ

メモ付きの記事には ✎ アイコンが表示され、カード上にメモのプレビューも表示されます。

### メモをまとめて見る

サイドバーの「**メモ一覧**」から、メモ付き記事を一覧で確認できます。
読書ノートや調査メモをまとめて振り返るのに便利です。

## 6. キーワードで自動収集する

Read Later の強力な機能のひとつです。キーワードを登録すると、Google Newsから関連ニュースを自動で見つけて保存します。

### 設定方法

1. サイドバーの「**設定**」を開く
2. 「**キーワード**」セクションで気になるワードを入力
3. 「**追加**」をタップ

### 例

```
Next.js
生成AI
量子コンピュータ
```

### 仕組み

- **30分ごとに自動チェック**されます
- マッチした記事はキーワード名のタグが自動で付きます
- 同じ記事が重複して保存されることはありません
- 「**今すぐチェック**」ボタンで手動実行もできます

## 7. RSSフィードを登録する

お気に入りのサイトの新着記事を、キーワードで自動フィルタリングして保存できます。

### 設定方法

1. サイドバーの「**設定**」を開く
2. 「**RSSフィード**」セクションでフィード名とURLを入力
3. 「**追加**」をタップ

### URLの入力について

**サイトのURLをそのまま入力してOKです**。RSSフィードURLが自動検出されます。

```
https://zenn.dev          → 自動で https://zenn.dev/feed を検出
https://example.blog.jp   → 自動でRSSを検出
```

もちろんRSS URLを直接入力することもできます：

```
https://zenn.dev/feed
https://example.com/rss.xml
```

### キーワードとの連携

RSSフィードから取得した記事のうち、**登録済みキーワードにマッチしたものだけ**が保存されます。
キーワードを先に登録しておいてください。

## 8. AIダイジェストを使う

登録したキーワードに関する今日のニュースを、AIが要約してくれます。

### 使い方

1. サイドバーの「**ダイジェスト**」を開く
2. 「**今すぐ生成**」をタップ
3. AIがキーワードごとのニュース要約を生成します

### 特徴

- **当日のニュースのみ**を対象にします
- 各ニュースの**出典メディアとリンク**が明記されます
- キーワードごとにセクション分けされた読みやすい形式
- 過去のダイジェストは日付タブで振り返れます

### 自動生成

毎朝7時（日本時間）に自動生成されます。
朝アプリを開くだけで、今日のニュースの概要がわかります。

## 9. AIリサーチを使う

特定のトピックについて調べたいとき、AIがニュースを検索して分析レポートを作ります。

### 使い方

1. サイドバーの「**リサーチ**」を開く
2. 調べたいことを入力欄に書く
3. **Cmd + Enter**（Mac）または **Ctrl + Enter**（Windows）で検索

> **注意**: 通常のEnterでは検索されません。Cmd/Ctrl + Enterを使ってください。

### 例

```
日本のAI規制の動向
React Server Componentsの最新状況
電気自動車の販売動向 2026
```

### 特徴

- 最新のGoogle Newsから最大15件の記事を検索
- AIが記事を分析し、**ソース付きのレポート**を生成
- レポート末尾に**参考文献リスト**（リンク付き）を表示
- **参考にした記事は自動で記事一覧に追加**されます（検索ワードのタグ付き）
- 過去のリサーチ履歴はいつでも参照できます

## 10. スマートフォンから使う

### iOSショートカットの設定（Safari共有ボタン対応）

iPhoneのSafariや他のアプリから、共有ボタンで直接記事を保存できます。

#### 準備：APIトークンを確認する

1. Read Later にログイン
2. サイドバーの「**設定**」を開く
3. 「**APIトークンを表示**」をタップ
4. 表示されたトークンをコピー

#### ショートカットを作成する

1. iPhoneの「**ショートカット**」アプリを開く
2. 右上の **+** をタップ
3. 「**アクションを追加**」→「**Web**」→「**URLの内容を取得**」を選択

4. 以下のように設定：

   | 項目 | 設定 |
   |------|------|
   | **URL** | `https://あなたのドメイン/read-later/api/articles` |
   | **方法** | POST |

5. 「**表示を増やす**」をタップして詳細を開く

6. **ヘッダ**に2つ追加：

   | キー | テキスト |
   |------|---------|
   | Content-Type | application/json |
   | Authorization | Bearer あなたのAPIトークン |

7. **本文を要求** → **JSON** を選択

8. キーと値を追加：

   | キー | 値 |
   |------|-----|
   | url | **ショートカットの入力** を選択 |

9. 共有シートの設定：
   - 画面上部の **ⓘ** をタップ
   - 「**共有シートに表示**」を **ON**
   - 入力タイプを「**URL**」のみに設定

10. 名前を「**Read Later**」に変更して保存

#### 使い方

Safariで記事を見ているとき：
1. 共有ボタン（□に↑）をタップ
2. 「**Read Later**」を選択
3. 自動で保存されます

## 11. 画面の見方

### PC（デスクトップ）

```
┌──────────┬──────────────────────────────────┐
│          │  [URLを追加...]  [追加] [新着|注目] │
│ サイドバー │──────────────────────────────────│
│          │                                  │
│ ・記事一覧 │  記事カード                       │
│ ・ダイジェスト│  記事カード                       │
│ ・リサーチ │  記事カード                       │
│ ・メモ一覧 │  ...                             │
│ ・設定    │                                  │
│          ├──────────────────────────────────│
│ 日付     │  [すべて] [未読] [お気に入り] [タグ]  │
│ ・今日    └──────────────────────────────────┘
│ ・昨日
│ ・3/28(土)
└──────────┘
```

- **サイドバー**は常に表示され、ページを切り替えてもそのまま残ります
- **日付で絞り込み**でその日に保存した記事だけを表示できます
- **下部のタブ**で未読・お気に入り・タグのフィルタリングができます

### スマートフォン

- サイドバーは左上の **☰** ボタンで開きます
- 下部にタブバー（すべて・未読・お気に入り・タグ）があります
- 指で操作しやすい大きめのボタンサイズです

## 12. よくある質問

### Q: 記事の本文が表示されないのはなぜ？

一部のサイトは本文の取得をブロックしています。その場合、「**読む**」ボタンの代わりに元記事へのリンクが表示されます。

### Q: キーワード自動収集が動いていないようです

- 設定画面でキーワードが登録されているか確認してください
- 「**今すぐチェック**」ボタンで手動実行してみてください
- 自動チェックは30分間隔です。少し待ってみてください

### Q: ダイジェストが生成されません

- キーワードが1つ以上登録されている必要があります
- 当日のニュースがない場合は生成されません
- 「**今すぐ生成**」ボタンで手動生成してみてください

### Q: 同じ記事が重複して保存されますか？

同じURLの記事は重複保存されません。キーワード自動収集でもリサーチでも、既に保存済みの記事はスキップされます。

### Q: パスワードを忘れました

現在、パスワードリセット機能はありません。サーバー管理者に連絡してください。

### Q: 注目度スコアはどう計算されますか？

以下の3つを組み合わせたスコアです：
- **はてなブックマーク数** — そのまま加算
- **Hacker News ポイント** — ×0.5で加算
- **ドメイン信頼度** — 官公庁(go.jp)は+50〜60、大学(ac.jp)は+20、PR Timesは+15など

### Q: データはどこに保存されますか？

すべてのデータはサーバー上のPostgreSQLデータベースに保存されます。外部サービスには送信されません（AIダイジェスト・リサーチ利用時のみ、記事タイトルがClaude APIに送信されます）。
